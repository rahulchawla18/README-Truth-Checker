# Drift Diagnoser Skill

The runner produces a verdict per step (pass / fail / skipped). For each failure, we want to answer: **what actually changed in the project that the README doesn't reflect?**

## Confidence levels

- **high** ÔÇö Direct evidence ties the failure to the codebase (the env var named in the `KeyError` appears in `os.environ[...]` in real code).
- **medium** ÔÇö Inferred but not certain (port detected, but the project might use a different one in some configs).
- **low** ÔÇö Generic failure, the suggestion is a hypothesis. Always show the user before any automated fix.

## Pattern catalog

### 1. Missing env var
**Trigger:** `KeyError: 'NAME'` in stderr/stdout, or `"NAME is not set" / "NAME must be set" / "NAME required"` messages.

**Confirm by reading the repo:**
- Search `*.py` for `os.environ["NAME"]`, `os.environ.get("NAME")`, `os.getenv("NAME")`, `getenv("NAME")`. Match must be the same NAME.
- Check `.env.example` / `.env.sample` / `.env.template` for a `NAME=` line.
- Run `git log -S NAME -- <file>` to find when the variable was introduced (useful evidence in the PR description).

**Fix:** Insert before the failing step:
```bash
# Set NAME in your .env file (see .env.example)
export NAME=<your-value>
```

### 2. Missing module after install (stale install command)
**Trigger:** `ModuleNotFoundError: No module named 'X'` AFTER the install step exited 0.

**Confirm by reading the repo:**
- `pyproject.toml` exists but README uses `pip install -r requirements.txt`? Ôćĺ mismatch.
- `poetry.lock` present but README uses bare `pip install`? Ôćĺ project moved to Poetry.
- `uv.lock` present but README uses `pip install`? Ôćĺ project moved to uv.

**Fix:** Swap the install command to match the actual tooling.

### 3. Wrong Python version
**Trigger:** Build fails with a SyntaxError on features that exist only in newer Python (match statement = 3.10, walrus = 3.8, f-strings = 3.6).

**Confirm by reading the repo:**
- `pyproject.toml` `requires-python` or `[tool.poetry.dependencies] python`
- `.python-version`
- `setup.py` `python_requires`

**Fix:** Update README's stated Python version requirement.

### 4. Wrong port (long-running step never ready)
**Trigger:** A long-running step started but the readiness probe on the README's port never succeeded. Or `Address already in use`.

**Confirm by reading the repo:**
- Search `manage.py`, `asgi.py`, `wsgi.py`, `main.py`, `app.py`, `config.py`, `settings.py` for `PORT = ...` or `port=...`.
- Fall back to framework defaults.

**Fix:** Update URLs (`localhost:OLD` Ôćĺ `localhost:NEW`) and any `--port OLD` flags in the README.

### 5. Missing migration step
**Trigger:** Django long-running step fails with `OperationalError: no such table` or similar. Alembic-backed app fails with "current revision is None."

**Confirm by reading the repo:**
- Django: `migrations/` directories exist with `0001_initial.py` files.
- Alembic: `alembic.ini` and `versions/` directory exist.

**Fix:** Insert `python manage.py migrate` (or `alembic upgrade head`) BEFORE the run step.

### 6. Renamed / moved entry point
**Trigger:** "command not found" or "No such file or directory" for a `.py` script.

**Confirm by reading the repo:**
- `rglob` for the script name ÔÇö is it in a subdirectory now?
- `git log -p -- README.md` to see if a previous version mentioned a different name.

**Fix:** Update the README's path (`python manage.py` Ôćĺ `python backend/manage.py`).

## What NOT to try to fix automatically

These patterns are diagnosed but only **reported**, never auto-patched:

- "unknown" failures (low confidence) ÔÇö surface the stderr excerpt and let the user judge.
- External service errors (DB connection refused, API rate limit) ÔÇö not README problems.
- Tests failing ÔÇö that's a code problem, not a README problem.

## Output discipline

Every drift point must include:
- `cause` (one of the categories above)
- `evidence` with file:line citations or git commits where possible
- `suggested_diff` as plain prose (the actual diff is generated by `generate_fix.py`)
- `confidence` (high / medium / low)
